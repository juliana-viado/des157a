By implementing these practices, product designers can improve user experience by adding elements that may not "belong" to spark curiosity and interaction to dig deeper.

Before reading this article, I took a look at the pictures for context. I am already subconciously trained to find meaning behind the mediums I absorb. I was confused by the collection of the images at first until I read the text. I would identify myself as a detail oriented, critical analyzer. Overlays Design Pattern Research

February 07, 2023

Naema Baskanderi's article, "Best Practices for Modals / Overlays / Dialog Windows" advocates for a better use of modal windows that avoids showing error, success, and warning information. By implementing these practices, product designers can improve the user flow.

User intitated actions, urgent messages, and popups for less important content is the way to go! Modals, overlays, and dialog windows should be used to grab attention, input user information, enlargement, and notifcations. The cancel button should be simple to find to return to main page by using the cancel button, close button, escape key, and clicking outside the window. Darken the background for a lightbox effect and don't use 50% of the screen for the overlay! Make sure the title is informative to and buttons have acionable names. It is good habit to avoid placing modals in mobile and use elements like the device keyboard and nested scrollbars.

Google Calendar is a an example of a modal. It is a user-intiated action. When editing a event, it uses a light box effect, clear buttons and user actions. I would improve it by making the buttons bigger.

Best Practices in Form Design

January 31, 2023

Salim Ansari's article, "Best Practices for Form Design" identifies necessary and unnecessary features to create optimal forms that limits friction and increases conversions. By implementing these practices, product designers can improve the ROI in our client's business.

Less is more! According to Baymard's user research, 21% of users abandon checkout because it is prolonged and/or complicated to finish the checkout process. A one column layout with a progress bar helps users visualize the number of steps before finishing and keeps them in the checkout process. For any follow up questions, save them at the end starting with the easiest! Make sure there is a clear distinction between optional and required information (especially if there are input requirements). It is good habit to avoid placing any text inside the form. Keep the inline labels above and the placeholder text below.

Gmail is a an example of a form website. They exhibit an error button with a error message "Couldn't find your Google Account" that has a red indicator under the form. Shop is easy to use because the tasks are simple. I would improve it by changing the "Next" button to "Sign In".
